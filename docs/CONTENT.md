コンテンツの実データは `src/content/`、型は `src/domains/` に置く。

## 2. サービス（5 本）

| サービス       | 概要                                                                                                                          |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| 受託開発       | 要件定義からフロント・バックエンド・クラウドインフラまで一貫。toC / toB / 社内システム                                        |
| ラボ型開発     | 専任チームによる継続的な開発・保守体制                                                                                        |
| テスト自動化   | Playwright による E2E / VRT を CI/CD に組み込み。AI でテスト生成を加速                                                        |
| AI駆動開発     | AI 駆動開発の基盤構築とコンサルティング（ワークフロー分析・ツール選定・Claude Code / MCP / CI 統合・規約 / スキル整備・教育） |
| 運用監視自動化 | エラー検知・Slack 通知と生成 AI による一次調査を自動化する運用監視基盤の構築                                                  |

- テスト自動化には**匿名の導入実例**（薬局向け LINE アプリの E2E 自動化で手動 QA 約 66% 削減）を掲載する。顧客名・サービス名は伏せる。
- 運用監視自動化には**実名の導入事例**（etoe sauna & hotel のエラー検知・AI 一次調査自動化）を掲載し、Works の `etoe` と相互リンクする。

## 3. 開発事例（Works・独立 3 件）

| slug                   | 公開区分 | 概要                                                                            |
| ---------------------- | -------- | ------------------------------------------------------------------------------- |
| `etoe`                 | 実名公開 | etoe sauna & hotel（etoehotel.com）の予約・施設管理 PMS をフルスタック内製      |
| `large-scale-platform` | 匿名     | 大規模基幹システムの継続開発・モダナイズ（14 年・約 4 万コミット・延べ 183 名） |
| `d2c-multi-brand`      | 匿名     | D2C 事業者の複数ブランド横断の開発・業務自動化 継続支援                         |

### 匿名化ポリシー（重要）

- `etoe` のみ事業名・公開 URL を掲載してよい。
- 匿名 2 件は**社名・サービス名・ブランド名・固有ドメインを掲載しない**。
- 匿名 2 件は**業種が特定される語を本文・技術タグに含めない**（例: 「広告」「SSP」「通信」「WiFi」）。一般名詞（「大規模 Web プラットフォーム」「D2C 事業者」）で表現する。
- 技術スタックと規模指標（年数・コミット数・人数）は開示してよい。
- 上記は `src/content/works.test.ts` でガードする。

## 5. お知らせ（News）

- microCMS は廃止し、記事を `src/content/news/<slug>.md`（YAML frontmatter + Markdown 本文・ファイル名 = slug）で静的に管理する。
  - frontmatter の日付は YAML の自動日付化を避けるためクォートする（例: `publishedAt: '2023-05-12'`）。
  - ビルド時に `scripts/generate-news-data.mjs` が `.md` を parse して `src/content/news.generated.json`（生成物・git 管理外）へ変換する。`src/content/news.ts` はこの JSON を import し、公開日の降順で返す。**Cloudflare Workers には src の `.md` が存在せず実行時 `fs` が使えない**ため、ビルド時にバンドルへインライン化する方式を採る（実行時 fs 依存ゼロ）。
  - codegen は `dev` / `build` / `build:dryrun` / `test`（`pnpm content:news`）の前段で自動実行される。`pnpm build` を経由する OpenNext の `preview` / `deploy` でも走る。
- カテゴリ: お知らせ / プレスリリース / イベント / メディア。
- 詳細ページ（`/news/[slug]`）は本文 Markdown を `Markdown` コンポーネント（react-markdown + remark-gfm）で描画する。本文中の外部リンク（`http(s)`）は別タブ（`target="_blank"` / `rel="noopener noreferrer"`）で開く。全記事が内部詳細ページを持つ（外部リンクのみで完結する記事は設けない）。
- 掲載内容（公開日の降順）:

| 公開日    | カテゴリ       | 見出し                                                          | リンク   |
| --------- | -------------- | --------------------------------------------------------------- | -------- |
| 2026/6/18 | お知らせ       | コーポレートサイトをリニューアルしました                        | 内部詳細 |
| 2023/6/27 | イベント       | "Engineer Driven Day 2023" にスポンサーとして参加します         | 内部詳細 |
| 2023/5/12 | メディア       | オンライン動画授業・講座サービス Schoo に講師として登壇しました | 内部詳細 |
| 2023/5/8  | お知らせ       | 事業拡大に伴いオフィスを移転しました                            | 内部詳細 |
| 2021/4/23 | プレスリリース | toraco株式会社を設立しました                                    | 内部詳細 |

- 記事本文は Markdown で記載する（見出し・箇条書き・強調・リンク等）。Schoo 記事は本文内に schoo.jp（講座ページ）への外部リンクを内包する。
- メタディスクリプションは本文 Markdown から記法を除いたプレーンテキスト（`src/utils/markdown.ts` の `toPlainText`）の先頭 80 文字を用いる。
